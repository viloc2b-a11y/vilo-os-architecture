# Vilo OS Core Architecture

## Objective

Define the final executable architecture for Vilo OS as an execution-native clinical research operating system centered on coordinator runtime orchestration.

## Non Goals

- Generic CTMS
- Sponsor-first architecture
- Enterprise bureaucracy
- Reporting-first systems
- AI wrapper products
- Dashboard-centric workflows

## Primary Runtime Model

Protocol → Visit → Procedure → Operational Event → Computed State

## Runtime Principles

- Visits are the atomic operational unit
- Protocols become executable runtime intelligence
- Governance emerges continuously
- Finance derives from operational truth
- Risk derives from execution signals
- TMF derives from lineage

## Core Layers

1. Execution Runtime Core
2. Protocol Intelligence Layer
3. Computation Fabric
4. Governance Fabric
5. Financial Intelligence Layer
6. Experience Layer

## MVP Scope

- Study runtime
- Subject runtime
- Visit orchestration

- eSource capture
- Safety orchestration
- Operational event spine
- Revenue traceability

## Deferred Scope

- Sponsor enterprise tooling
- Marketplace integrations
- CRO workflow systems
- Multi-tenant enterprise abstractions
