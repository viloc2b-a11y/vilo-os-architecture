\# Protocol Graph Engine



Status: Draft



\---



\# Objective



Define the protocol intelligence engine that converts protocol requirements into executable runtime logic for visits, procedures, dependencies, windows, conditional workflows, safety triggers, and amendment impact.



\---



\# Core Principle



A protocol is not a PDF.

A protocol is an executable operational graph.



\---



\# Responsibilities



The Protocol Graph Engine must define:



\- visit definitions

\- procedure definitions

\- visit windows

\- required vs conditional procedures

\- dependency rules

\- safety triggers

\- off-site eligibility

\- PK logic

\- amendment deltas

\- source requirements

\- financial implications



\---



\# Core Graph Nodes



\- Study

\- Protocol Version

\- Visit Definition

\- Procedure Definition

\- Procedure Requirement

\- Eligibility Rule

\- Safety Rule

\- Dependency Rule

\- Source Template

\- Billable Mapping

\- Amendment Delta



\---



\# Core Graph Edges



\- protocol\_requires\_visit

\- visit\_requires\_procedure

\- procedure\_depends\_on\_procedure

\- result\_triggers\_workflow

\- event\_generates\_billable

\- amendment\_changes\_requirement

\- procedure\_requires\_source

\- safety\_event\_blocks\_visit

\- lab\_result\_triggers\_action



\---



\# Runtime Outputs



The engine must produce:



\- executable visit schedule

\- procedure checklist

\- visit readiness rules

\- source package requirements

\- conditional workflow triggers

\- deviation rules

\- financial expected billables

\- operational risk signals



\---



\# MVP Scope



MVP must support:



\- static visit definitions

\- required procedure mapping

\- visit windows

\- procedure dependencies

\- conditional procedure flags

\- source template linkage

\- expected billable generation

\- amendment versioning placeholder



\---



\# Deferred Scope



Deferred:



\- full NLP protocol parser automation

\- autonomous protocol interpretation

\- sponsor portal configuration

\- enterprise rule marketplace

\- complex visual graph editor



\---



\# AI Usage



AI may assist with:



\- protocol extraction

\- schedule of activities parsing

\- procedure normalization

\- risk flagging

\- draft graph generation



AI must not autonomously publish protocol logic without human review.



\---



\# Final Rule



If protocol complexity is pushed back to the coordinator, the engine failed.
